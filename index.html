Configuração do Firebase

Cole aqui a config do seu projeto Firebase (Firestore + Storage + Authentication com e-mail/senha habilitado). Isso fica salvo só no seu navegador (localStorage) — não sobe pra lugar nenhum.

Importante — regras de segurança: pra que o login realmente proteja os dados, você precisa colar isto no Firebase Console, em Firestore Database → Regras: 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
E em Storage → Regras: 
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if request.auth != null;
    }
  }
}
Sem isso, o login aparece na tela mas os dados continuam abertos pra qualquer um com a config do projeto.

Chave do Gemini

A chave do Gemini agora fica só no servidor (variável de ambiente no Vercel) — o app nunca vê nem envia essa chave. Ele chama /api/gemini, que confere seu login e repassa a chamada. Não precisa configurar nada aqui.

Como funciona

  1. Cada pessoa autorizada cria sua conta (e-mail/senha) na tela de login, uma única vez.
  2. Importe o extrato em PDF do mês — o Gemini lê o PDF e extrai as transações (revise antes de confirmar).
  3. Tire ou suba fotos dos comprovantes — pode ter mais de um comprovante por foto. O Gemini identifica cada um; você revisa e confirma antes de salvar.
  4. O sistema tenta conciliar automaticamente por valor + data. O que não bater, fica pendente.
  5. Na aba Conciliação, você vincula manualmente o que sobrou, ou justifica a ausência de comprovante.
  6. Em Exportar, você gera a planilha do mês + baixa os comprovantes em lote pra entregar ao juiz.
